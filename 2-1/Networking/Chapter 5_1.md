### 1\. Per-Router Control (Traditional)

In this approach, the routing algorithm is **distributed**. Each router in the network runs its own routing algorithm component.

+1

-   **Interaction:** Individual routing components in every router interact with one another by exchanging messages.
    
-   **Local Calculation:** Each router calculates its own contribution to the global routing path.
    
-   **Result:** The algorithm produces a **local forwarding table**, which the data plane then uses to move packets.
    
-   **Protocols:** Common examples include OSPF (Open Shortest Path First) and BGP (Border Gateway Protocol).
    

---

### 2\. Logically Centralized Control (SDN)

In **Software-Defined Networking (SDN)**, the routing logic is removed from individual routers and moved to a **remote, centralized controller**.

-   **The Controller:** A central server (the SDN controller) calculates the paths for the entire network.
    
    +1
    
-   **Control Agents (CA):** Each router has a Control Agent that communicates with the remote controller.
    
-   **Installation:** The controller computes the forwarding tables and "installs" them into each router's data plane.
    
    +1
    
-   **Efficiency:** This allows for more "generalized forwarding," where routers can make decisions based on many header fields, not just the destination IP.

### Key Comparison

| Feature | Per-Router Control | Logically Centralized (SDN) |
| --- | --- | --- |
| **Logic Location** | Distributed in every router +1 | Centralized in a remote server |
| **Decision Maker** | Individual routing algorithms | SDN Controller |
| **Inter-device comms** | Routers talk to each other | Routers talk to the Controller |
